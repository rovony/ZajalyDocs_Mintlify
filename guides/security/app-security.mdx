Application security starts with validating and sanitizing all user input and hardening your HTTP responses.

<Danger>Never trust user input. Validate server-side, sanitize before rendering, and escape before storing. Assume all input is malicious until proven safe.</Danger>

## Common vulnerabilities

<AccordionGroup>
  <Accordion title="XSS (Cross-Site Scripting)" icon="code">
    **Cause:** Rendering unsanitized user input in HTML or JavaScript.  
    **Fix:** Escape output (e.g. HTML entities) and use Content-Security-Policy.  
    **Example:** Use a sanitizer like `DOMPurify` in frontend; `htmlspecialchars()` in PHP.
  </Accordion>
  <Accordion title="CSRF (Cross-Site Request Forgery)" icon="shield-halved">
    **Cause:** Attacker tricks a user's browser into submitting requests to your app.  
    **Fix:** Use CSRF tokens, SameSite cookies, and verify Origin/Referer headers.  
    **Example:** Laravel `@csrf`; Express `csurf` middleware.
  </Accordion>
  <Accordion title="SQL Injection" icon="database">
    **Cause:** Concatenating user input into SQL strings.  
    **Fix:** Use parameterized queries or an ORM. Never build SQL with string concatenation.  
    **Example:** `$pdo->prepare('SELECT * FROM users WHERE id = ?')->execute([$id]);`
  </Accordion>
  <Accordion title="Rate limiting" icon="clock">
    **Cause:** No limits on requests; enables brute force and DoS.  
    **Fix:** Limit requests per IP/user (e.g. 100/min for API, 5/min for login).  
    **Example:** Laravel `throttle:60,1`; Nginx `limit_req_zone`; Express `express-rate-limit`.
  </Accordion>
</AccordionGroup>

## Sanitization examples

<CodeGroup>
```javascript Node.js
const createDOMPurify = require('dompurify');
const { JSDOM } = require('jsdom');
const purify = createDOMPurify(new JSDOM('').window);
const safe = purify.sanitize(userInput, { ALLOWED_TAGS: ['b', 'i'] });
```

```php Laravel
{{-- Blade auto-escapes --}}
{{ $user->name }}

{{-- Raw (dangerous unless sanitized) --}}
{!! strip_tags($html) !!}
```
</CodeGroup>

<Tip>Add security headers: `Content-Security-Policy`, `X-Content-Type-Options: nosniff`, `X-Frame-Options: DENY`, `Strict-Transport-Security`. Use middleware or server config.</Tip>

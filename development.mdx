## Preview locally

Navigate to your docs directory where `docs.json` is located:

```bash
mint dev
```

A local preview will be available at `http://localhost:3000`.

<Check>You should see the Zajaly Developer Docs home page in your browser.</Check>

## Custom ports

By default, Mintlify uses port 3000. Customize with the `--port` flag:

```bash
mint dev --port 3333
```

If the port is in use, the CLI auto-selects the next available port.

## CLI commands

| Command | Purpose |
|---|---|
| `mint dev` | Local preview at localhost:3000 |
| `mint update` | Update CLI to latest version |
| `mint broken-links` | Find broken internal links |
| `mint a11y` | Check accessibility |
| `mint validate` | Validate build in strict mode |
| `mint rename <old> <new>` | Rename file and update references |

## Troubleshooting

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module'>
    This may be due to an outdated version of Node.js. Try the following:
    1. Remove the currently-installed version of the CLI: `npm remove -g mint`
    2. Upgrade to Node v20.17.0 or higher.
    3. Reinstall the CLI: `npm i -g mint`
  </Accordion>
  <Accordion title="Encountering an unknown error">
    Go to the root of your device and delete the `~/.mintlify` folder. Then run `mint dev` again.
  </Accordion>
  <Accordion title="Local preview doesn't match production">
    Run `mint update` to get the latest CLI version. The local CLI version must match the production build version.
  </Accordion>
</AccordionGroup>

Curious about what changed in the latest CLI version? Check out the [CLI changelog](https://www.npmjs.com/package/mintlify?activeTab=versions).

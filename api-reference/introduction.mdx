The Zajaly API is organized around <Tooltip tip="Representational State Transfer — an architectural style for designing web APIs">REST</Tooltip>. It accepts JSON request bodies, returns JSON responses, and uses standard HTTP status codes.

## Base URL

All API requests are made to:

```
https://api.zajaly.dev/v1
```

## Authentication

All endpoints require a Bearer token in the `Authorization` header.

<CodeGroup>

```bash cURL
curl -X GET 'https://api.zajaly.dev/v1/health' \
  -H 'Authorization: Bearer YOUR_API_KEY'
```

```javascript Node.js
const response = await fetch('https://api.zajaly.dev/v1/health', {
  headers: { Authorization: 'Bearer YOUR_API_KEY' }
});
```

```python Python
import requests

response = requests.get(
    'https://api.zajaly.dev/v1/health',
    headers={'Authorization': 'Bearer YOUR_API_KEY'}
)
```

</CodeGroup>

<Check>A successful health check returns `{"status": "ok"}`.</Check>

See the [Authentication](/api-reference/authentication) page for details on obtaining and managing API keys.

## Status codes

| Code | Meaning |
|---|---|
| `200` | Request succeeded |
| `201` | Resource created |
| `400` | Invalid request parameters |
| `401` | Missing or invalid API key |
| `404` | Resource not found |
| `429` | Rate limit exceeded |
| `500` | Server error |

## Rate limits

API requests are limited to **100 requests per minute** per API key. When exceeded, you receive a `429` response with a `Retry-After` header.

<Warning>Exceeding rate limits repeatedly may result in temporary key suspension.</Warning>
